# Electron 缓存错误解决方案

## 错误信息
```
ERROR:cache_util_win.cc(20)] Unable to move cache: 拒绝访问。 (0x5)
ERROR:disk_cache.cc(208)] Unable to create cache
ERROR:gpu_disk_cache.cc(676)] Gpu Cache Creation failed: -2
```

## 原因
这些错误通常由以下原因引起：
1. **缓存目录权限问题** - Windows 对某些目录的访问限制
2. **GPU 加速冲突** - 硬件加速与系统不兼容
3. **缓存文件损坏** - 上次异常退出导致缓存损坏
4. **防病毒软件拦截** - 实时保护阻止缓存写入

## 解决方案

### 方案 1：使用清理脚本（推荐）

#### 步骤 1：清理旧缓存
双击运行 `clear-cache.bat`，脚本会自动删除以下目录：
- `%LOCALAPPDATA%\Electronic\Cache`
- `%LOCALAPPDATA%\Electronic\GPUCache`
- `%LOCALAPPDATA%\Electronic\Code Cache`

#### 步骤 2：无缓存启动
双击运行 `start-without-cache.bat`，此脚本会：
- 设置环境变量禁用 GPU
- 设置环境变量禁用软件光栅化器
- 启动应用

### 方案 2：手动清理

1. 打开运行对话框（Win + R）
2. 输入 `%LOCALAPPDATA%\Electronic`
3. 删除以下文件夹（如果存在）：
   - `Cache`
   - `GPUCache`
   - `Code Cache`
   - `ShaderCache`
   - `IndexedDB`
4. 重新启动应用

### 方案 3：修改启动参数（开发者）

在 `package.json` 中修改启动脚本：

```json
{
  "scripts": {
    "electron": "vite --mode electron -- --no-sandbox --disable-gpu --disable-software-rasterizer"
  }
}
```

### 方案 4：在应用中禁用硬件加速（已自动应用）

应用已自动设置：
- `app.disableHardwareAcceleration()` - 禁用硬件加速
- `cache: false` - 禁用缓存
- `partition: 'persist:electronic-main'` - 使用独立分区

### 方案 5：添加防病毒软件例外

如果使用 Windows Defender：
1. 打开 Windows 安全中心
2. 病毒和威胁防护 → 管理设置
3. 排除项 → 添加或删除排除项
4. 添加 `%LOCALAPPDATA%\Electronic` 为排除文件夹

如果使用第三方防病毒软件：
1. 打开防病毒软件设置
2. 添加应用目录为信任列表
3. 添加缓存目录为信任列表

### 方案 6：检查文件权限

1. 右键 `%LOCALAPPDATA%\Electronic` 文件夹
2. 属性 → 安全
3. 确保当前用户有完全控制权限
4. 如无，点击"编辑"并添加当前用户

## 临时解决方案（开发环境）

如果上述方案都无效，可以尝试在开发模式下禁用某些功能：

在 `electron/index.js` 中已有以下配置：

```javascript
// 自动清理旧缓存
const cleanOldCache = () => {
  try {
    const userDataPath = app.getPath('userData')
    const cachePath = join(userDataPath, 'Cache')
    if (fs.existsSync(cachePath)) {
      fs.rmSync(cachePath, { recursive: true, force: true })
      console.log('[Main] 旧缓存已清理')
    }
  } catch (error) {
    console.log('[Main] 清理缓存失败（忽略）:', error.message)
  }
}

// 启动时自动清理
cleanOldCache()

// 禁用 GPU
app.commandLine.appendSwitch('disable-gpu')
app.commandLine.appendSwitch('disable-software-rasterizer')
```

## 验证修复

启动应用后，查看控制台日志：
- ✓ 不应再看到缓存错误
- ✓ 应用应正常加载
- ✓ 功能应正常使用

## 如果问题仍然存在

1. **完全重置应用数据**：
   ```
   删除 %LOCALAPPDATA%\Electronic 整个文件夹
   重新启动应用（会丢失所有设置和数据）
   ```

2. **以管理员身份运行**：
   ```
   右键应用 → 以管理员身份运行
   ```

3. **联系开发者**：
   - 提供完整错误日志
   - 说明操作系统版本
   - 说明是否使用了防病毒软件
